# 徳島県 クマ出没ハザードマップ 要件定義書

## 1. 背景・目的
徳島県のツキノワグマ出没記録は約20件と非常に少ない。出没点をそのまま
点密度（KDE）でヒートマップ化しても「過去に人が偶然遭遇した場所」が
光るだけで、実際に危険な箇所を表現できない。

本プロジェクトは **「地形・土地利用などの地理的特徴が、既知の出没地点と
似ている場所」** をスコア化し、徳島県全域のリスクをヒートマップとして
可視化する **生息適性（ハザード）推定** を行う。

データは presence（出没）のみで真の不在データが無いため、
presence-only な特徴空間類似度モデルを採用する。

## 2. 利用者・利用シーン
- 登山者・林業従事者・観光客・住民が、入山前にエリアの相対リスクを確認する
- 自治体・研究者が、出没傾向を地理的に俯瞰する

## 3. 機能要件
| ID | 内容 |
|----|------|
| FR1 | 出没記録20件を緯度経度・痕跡種別・信頼度付きデータとして整備する |
| FR2 | 徳島県全域を標準地域メッシュ（3次メッシュ ≒ 1km）でグリッド化する |
| FR3 | 各メッシュに地理的特徴量を付与する（標高・傾斜・起伏・森林率・建物用地率・農地率・最近隣河川距離） |
| FR4 | 出没メッシュの特徴分布を基準に、各メッシュの類似度＝ハザードスコア(0–1)を算出する |
| FR5 | スコアをメッシュ色分け（ヒートマップ）でWeb地図に表示する |
| FR6 | 出没点をマーカー表示し、ポップアップで日付・場所・状況・痕跡種別を確認できる |
| FR7 | レイヤ切替（ハザード／出没点）と凡例・出典表記を備える |

## 4. 非機能要件
| ID | 内容 |
|----|------|
| NFR1 | 前処理は `python src/pipeline.py` で生データ→成果物まで再現可能 |
| NFR2 | Webマップは静的ファイル（HTML/JS/GeoJSON）のみで動作し GitHub Pages で配信可能 |
| NFR3 | 生データ（KSJ/DEM）は容量大のため Git 管理外。成果物 GeoJSON はコミットし即表示可能にする |
| NFR4 | データの出典・ライセンスを README に明記（国土数値情報・国土地理院は出典表示が必要） |

## 5. スコープ外
- 時系列・季節変動予測、高度な機械学習チューニング
- リアルタイムデータ連携、通報投稿機能、アプリ化

## 6. データソース
| 用途 | データ | 提供 |
|------|--------|------|
| 標高・傾斜・起伏 | 数値標高モデル DEM10B（標高タイル） | 国土地理院 |
| 土地利用（森林率・建物用地率・農地率） | 土地利用細分メッシュ L03-b | 国土数値情報 |
| 河川（最近隣距離） | 河川データ W05 | 国土数値情報 |
| 森林地域（参考） | 森林地域 A45 | 国土数値情報 |
| 県境ポリゴン（グリッド切り抜き） | 行政区域 N03 | 国土数値情報 |

対象都道府県コード：36（徳島県）／対象1次メッシュ：5033, 5034, 5133, 5134

## 7. 手法（FR4 の核）
presence-only 特徴空間類似度モデル：
1. 各メッシュの特徴ベクトル `x = [標高, 傾斜, 起伏, 森林率, 建物用地率, 農地率, log(河川距離)]`
2. 全メッシュ統計で z-score 標準化
3. 全メッシュ（背景）から正則化付き共分散 Σ を推定
4. 各メッシュのスコア `= mean_i exp( −0.5 · d_M(x, p_i)² / h² )`
   （p_i：出没メッシュ、d_M：マハラノビス距離、h：バンド幅）
5. スコアを 0–1 に正規化して GeoJSON に格納
- 妥当性確認：leave-one-out による出没メッシュの上位パーセンタイル

## 8. 成果物
- `data/sightings.csv`：整備済み出没記録
- `data/processed/grid_scores.geojson`：メッシュ＋ハザードスコア
- `data/processed/sightings.geojson`：出没点
- `web/`：Leaflet Webマップ（GitHub Pages 公開）

## 9. 検証
- `python src/pipeline.py` が成果物 GeoJSON を生成する
- score が 0–1 で、出没メッシュが高スコア側に分布（leave-one-out で確認）
- `web/index.html` をローカル配信し、ヒートマップ・出没点・凡例・レイヤ切替が表示される
- GitHub Pages URL で表示される

## 10. 注意・免責
本マップは少数データに基づく統計的推定であり、スコアが低い場所の安全を
保証するものではない。クマの生息域では常に十分な注意が必要である。
